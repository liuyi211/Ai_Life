## 3. 技术约束与规范

### 3.1 代码规范

```
- 所有代码必须通过 TypeScript 严格模式检查
- 禁止使用 any 类型（特殊情况需注释说明）
- 禁止使用 @ts-ignore
- 组件必须使用函数式组件 + Hooks
- 每个模块必须包含单元测试（覆盖率 >= 80%）
- 提交信息遵循 Conventional Commits 规范
```

### 3.2 游戏数据规范

```
- 所有数值配置集中管理，使用配置文件
- 事件触发条件使用声明式配置
- 天赋效果使用纯函数实现，无副作用
- 存档数据版本化，支持向后兼容
```

### 3.3 AI提示词规范

```
- 提示词模板集中管理（prompts/目录）
- 使用结构化输出（JSON模式）
- 提示词包含角色上下文、历史摘要、当前状态
- 预留提示词调试接口
```

### 3.4 API规范

```
- RESTful 设计，使用标准 HTTP 方法
- 统一响应格式：{ success, data, message, error }
- 错误码全局统一管理
- 认证接口：/api/auth/*
- 用户接口：/api/users/*
- 存档接口：/api/saves/*
- AI接口：/api/ai/*
```

## 4. 风险与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| AI API响应慢/不稳定 | 高 | 高 | 实现流式输出+缓存+降级文本 |
| 用户模板延迟提供 | 中 | 中 | 先用基础UI开发逻辑，模板到后替换样式 |
| 数据库Schema频繁变更 | 中 | 中 | 使用Prisma Migration，版本化管理 |
| 游戏平衡性问题 | 中 | 中 | 预留数值调整接口+玩家反馈机制 |
| 多世界开发工作量大 | 高 | 中 | 优先完成地球世界，其他世界逐步扩展 |
