- [x] **[TASK-801]** 实现结算页面（SettlementPage.tsx）
  - **描述**: 角色死亡后展示人生总结、成就、评分的页面
  - **验收标准**: 
    - 展示角色一生关键事件时间线
    - 显示最终属性值和人生评价
    - 展示获得的成就和称号
    - 提供"进入轮回"按钮
  - **状态**: ✅ 已完成
  - **依赖**: 阶段07完成

- [x] **[TASK-802]** 实现后端结算API
  - **描述**: 处理角色死亡后的结算逻辑（评分、成就检查、命运碎片计算）
  - **验收标准**: 
    - `POST /api/saves/:id/settle` 接口可用
    - 根据角色一生数据计算综合评分
    - 检查并解锁符合条件的成就
    - 计算本世获得的命运碎片数量
  - **状态**: ✅ 已完成
  - **依赖**: TASK-801

- [x] **[TASK-803]** 实现轮回继承系统
  - **描述**: 新周目时根据上一世表现给予继承加成
  - **验收标准**: 
    - 基因继承：高属性有概率遗传（+1~+3初始属性）
    - 家族信托：上一世资产转化为命运碎片
    - 残卷继承：上一世获得的特殊物品/功法带入下一世
    - 创建新角色时自动应用继承效果
  - **状态**: ✅ 已完成
  - **依赖**: TASK-802

- **当前完成度**: 阶段01-08 全部完成，阶段09基础完成，阶段10优化项待开始
- **新增功能**: 存档模板系统（3个测试模板 + 快速模式）
- **关键里程碑**: 已实现完整的"注册→登录→创建角色→AI叙事游戏循环→死亡"端到端流程，支持页面刷新恢复
- **技术栈确认**: React 19 + Vite 8 + Express + Prisma + PostgreSQL
- **当前数据库状态**: 用户表、存档表、成就表、用户成就表 结构完整
- **下一步重点**: 
  1. 实现结算页面（SettlementPage.tsx）和结算API
  2. 实现轮回继承系统（基因+家族信托+残卷）
  3. 添加NPC对话界面（API已就绪）
  4. 实现资产/关系/技能面板展示
- **已知缺失（不影响核心流程）**:
  - 流式SSE输出（当前打字机效果已满足体验需求）
  - 用户资料管理（修改密码、头像）
  - 存档导入/导出
  - 音效和背景音乐
  - 响应式移动端适配
- **关键决策**: 
  - 存档数据使用 JSON 格式存储在 PostgreSQL 中（灵活，适合游戏状态）
  - 属性系统简化为4维（体魄/悟性/魅力/气运），降低复杂度
  - 本地备份采用 localStorage 双写策略（后端+本地），实现简单且满足需求
  - 流式输出改用客户端打字机效果，避免SSE复杂度
